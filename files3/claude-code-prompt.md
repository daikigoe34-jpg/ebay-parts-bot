# 手数料体系（正確に実装すること）

## eBay手数料（USDベースで天引き）
- FVF落札手数料: (商品価格+送料) × 13.25% + $0.40
  ※自動車パーツでストアありなら12.35%。config.pyで変更可能に
- 海外決済手数料: (商品価格+送料) × 1.35%
  ※ボリュームディスカウントあり。初心者は1.35%固定でOK
- eBay手数料合計 ≒ 売上の約14.6% + $0.40

## Payoneer為替手数料（JPY引き出し時）
- 為替レートに2%上乗せ（仲値の98%で換算）
- 実装: 実効為替レート = config.pyの為替レート × 0.98
- eBayからの受取額(USD) × 実効レート = 日本円受取額

## DDP関税（アメリカ宛・自動車部品）
- 商品申告価格 × 15%（MFN税率2.5%込み・日米合意ベース）
- SpeedPAK経由で別途請求される
- ※関税政策は流動的。ebay.co.jp/tariffs で最新確認

## SpeedPAK Economy送料（アメリカ宛）
- 重量帯テーブル: ~100g:¥1,227 / ~300g:¥1,500 / ~500g:¥1,800 / ~1kg:¥2,800 / ~1.5kg:¥3,500
- 容積重量計算: 縦×横×高さ(cm)÷8000、実重量と大きい方を適用

## その他
- 梱包材: ¥300固定
- 国内送料: ¥0（郵便局/ローソン持ち込み）

# 作ってほしいもの

## config.py
- 上記の全手数料パラメータを1箇所で管理
- 各パラメータに日本語コメント + 出典URL
- 「関税率・eBay手数料率は頻繁に変わる」の注意コメント

## ebay_client.py
- eBay Finding API `findCompletedItems` でsold items検索
- OAuth認証実装
- カテゴリ: Auto Parts & Accessories (6030)

## shipping.py
- SpeedPAK Economy送料テーブル（国別・重量帯別）
- 容積重量計算

## calculator.py ← ここが一番重要
- 上記の手数料体系を正確に実装
- 計算フロー:
  1. 販売価格(USD)からeBay手数料(FVF+海外決済)を引く → eBay受取額(USD)
  2. 受取額をPayoneer実効レート(仲値×0.98)でJPY変換 → 日本円受取額
  3. DDP関税(JPY)を別途計算
  4. 利益 = 日本円受取額 - DDP関税 - 仕入原価 - SpeedPAK送料 - 梱包材
- 各コストの内訳を辞書で返す（CSV出力用）

## main.py
- 全キーワード検索 → 利益計算 → 利益>0でフィルタ → 利益率順ソート
- コンソール表形式 + CSV出力（日付入りファイル名）
- サマリー表示: 検索数/黒字商品数/平均利益率/TOP5
- --demo フラグでダミーデータ実行（$40〜$100の自動車部品）

- 使い方、手数料テーブル更新方法、関税率変更方法

## 補足プロンプト

### ストア契約後にFVFを下げたいとき
```
eBay Basicストアに加入した。
config.pyのFVF_RATEを12.35%に変更して再計算して。
```

### 関税率が変わったとき
```
アメリカの自動車部品関税率が変わった。
config.pyのDDP_TARIFF_RATEをX%に更新して再計算。
最新情報: https://ebay.co.jp/tariffs
```

### Payoneerの為替手数料を正確に反映
```
Payoneerの実際の換算レートを確認したら仲値から1.5%だった。
config.pyのPAYONEER_FX_MARKUP を 0.015 に変更して。
```

### モノタロウ仕入価格CSV反映
```
仕入原価をCSVから読み込めるようにして:
商品名,モノタロウ品番,仕入価格,重量g,梱包後サイズcm(縦x横x高さ)
オイルフィルター,12345678,980,200,10x8x8
ブレーキパッド,23456789,3500,800,20x15x5
```

### イギリス向け（関税有利）
```
イギリス宛はDDP不要で関税0%。
--country uk オプションで切り替え可能にして。
イギリス向けSpeedPAK Economy送料: ~100g:¥938 / ~500g:¥XXXX / ~1kg:¥XXXX
```
